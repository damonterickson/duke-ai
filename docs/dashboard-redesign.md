## Problem Statement

The current dashboard is functional but visually basic — a simple OML gauge, flat pillar bars, and a plain activity log. The Stitch "Duke Vanguard" project has a polished dashboard design with a hero section, bento grid layout, AI insight card, and rich activity log. The app needs to match this design to feel production-quality.

## What's Changing

### Current Dashboard → Stitch Dashboard

| Element | Current | Stitch Target |
|---------|---------|---------------|
| Hero section | VConicGauge only | Gauge + "Mission Readiness Profile" text + AI Insight card (border-left accent) |
| Stat cards | 3 VProgressBars inline | 3 bento grid cards with icons, weight labels ("WEIGHT: 40%"), large score numbers |
| Activity log | Basic VActivityItem list | Rich items with 12px icon circles, colored point deltas, timestamps |
| Goals section | Active Goals (0/5) | Integrated between stat cards and activity log |
| FAB | Chat (olive gradient circle) | "New Simulation" (calculate icon + text label, rounded-xl) |
| Header | Tab bar only | App bar with logo + "Upload Accession Sheet" button |

### New UI Elements

1. **AI Insight Card** — border-left-4 accent in primary color, psychology icon (filled), "Vanguard AI Insight" label, recommendation text. Uses primary-container/10 background.

2. **Stat Cards (bento grid)** — Each card has:
   - Icon badge (Material icon in surface-container-high rounded-lg)
   - Weight label ("WEIGHT: 40%" in outline color)
   - Category name (headline bold)
   - Score display (3xl font-black + "/ max")
   - Progress bar (primary for academic, tertiary for physical, primary-container for leadership)

3. **"New Simulation" FAB** — gradient background (primary → primary-container), calculate icon + text label, rounded-xl, positioned bottom-right above tab bar.

4. **Upload Accession Sheet** — Button in header area (or accessible from dashboard). Opens the existing Upload & Sync screen.

## Constraints

- Must follow the "Modern Vanguard" design system (DESIGN.md)
- Reuse existing components where possible (VConicGauge, VProgressBar, VGoalCard, VActivityItem)
- Keep the goals section from the v1 build
- Dashboard FAB = "New Simulation" (→ What-If simulator). Chat FAB stays on Advisor tab only.
- No new SQLite tables or API changes — this is a UI-only update
- Must work offline (all data from local stores)

## Premises

1. The Stitch HTML is the visual target — match it as closely as React Native allows.
2. The dashboard is one screen — no need to extract single-use components.
3. The AI Insight card uses the last cached briefing — no new API call needed.
4. The "Upload Accession Sheet" button navigates to the existing upload screen (Wave 3 — may not be fully built yet, so it can be a placeholder).

### VInsightCard Component

```typescript
interface VInsightCardProps {
  icon: string;          // Material icon name
  label: string;         // "Vanguard AI Insight"
  text: string;          // The recommendation text
  accentColor?: string;  // defaults to primary
  style?: ViewStyle;
}
```

Design:
- Left border: 4px accent in accentColor
- Background: primary_container at 10% opacity
- Icon: filled variant, primary color
- Label: Inter uppercase, xs, bold, primary
- Text: primary color, lg, medium weight, leading-snug

### Stat Card Design (inline in dashboard.tsx)

Each card is a View with:
- surface_container_lowest background, rounded-xl
- Top row: icon badge (surface_container_high bg, rounded-lg) + weight label (outline color, uppercase)
- Title: headline bold xl
- Score: 3xl font-black on_surface + "/ max" in on_surface_variant
- VProgressBar with category-specific color

### "New Simulation" FAB

- Position: absolute, bottom-right (above tab bar)
- Background: LinearGradient (primary → primary_container)
- Icon: calculate
- Text: "New Simulation" in label font, bold, uppercase
- Rounded-xl (not rounded-full like the current FAB)
- onPress: router.push('/what-if')
