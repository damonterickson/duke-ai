## Problem Statement

Duke Vanguard tracks OML scores and shows optimization recommendations, but the AI's advice disappears into chat history. There's no way for a cadet to commit to a recommendation, track progress toward it, or see how their daily actions connect to OML improvement. The app shows where you are — but doesn't help you get where you want to be.

## What Makes This Cool

Two-way AI-native goals. The AI doesn't just suggest — it **coaches**. When you opt into AI Coach mode, the AI proactively creates goals based on your weakest pillar and highest marginal gains, adjusts them as your data changes, and celebrates when you hit them. Goals auto-update when you log new data — no manual progress tracking. The AI is your personalized drill sergeant, but supportive.

The "whoa" moment: you log a new ACFT score, and your goal card immediately updates from "530/550" to "545/550 — 5 points to go!" and the AI says "You're almost there. One more plank session closes the gap."

## Constraints

- Must work offline (goals stored in SQLite, progress updates are local)
- Max 5 active goals at a time (focus > ambition)
- Goals live on the Dashboard (Mission Profile becomes "mission command")
- Must integrate with existing OML engine marginal gains
- Must work with OpenRouter API (structured JSON output alongside chat text)
- Must follow the "Modern Vanguard" design system (No-Line Rule, gold for achievements)

## Premises

1. Goals need both a target value AND a deadline — without a deadline, a goal is just a wish.
2. Two modes: Manual (cadet creates goals) + AI Coach (opt-in, AI auto-creates and manages goals).
3. Goals auto-update when new data is logged (ACFT, GPA, leadership entries).
4. Completed goals celebrate with gold animation + OML impact shown.
5. Max 5 active goals at a time.

## Approaches Considered

### Approach A: Goals as Data Layer
Simple SQLite table, manual AI suggestions, deterministic progress tracking.
- Effort: M, Risk: Low
- Pros: Simple, reliable, works offline
- Cons: AI Coach feels bolted on, separate API call needed

### Approach B: AI-Native Goals (CHOSEN)
AI manages goals via structured JSON in every response. When AI Coach mode is on, the AI's system prompt includes goal management instructions. After every briefing or chat response, the AI returns JSON with goal actions (create/update/complete) alongside the text. The app parses and applies them automatically.
- Effort: M, Risk: Medium (LLM JSON reliability)
- Pros: Coach feels intelligent, goals adapt automatically, single API call
- Cons: Depends on LLM structured output
- Reuses: AI service, OML engine marginal gains, Context Engine, Dashboard

### Approach C: Notification-Driven Habits
Daily check-off habits instead of target-based goals.
- Effort: S, Risk: Low
- Cons: Disconnected from OML data, doesn't leverage the engine

## Recommended Approach

**Approach B: AI-Native Goals.**

### Data Model

```sql
CREATE TABLE goals (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,                    -- "Improve ACFT to 550"
  category TEXT NOT NULL,                 -- 'gpa' | 'acft' | 'leadership' | 'oml'
  metric TEXT NOT NULL,                   -- 'acft_total' | 'gpa' | 'leadership_eval' | 'total_oml'
  target_value REAL NOT NULL,             -- 550
  current_value REAL,                     -- 530 (auto-updated from latest data)
  baseline_value REAL NOT NULL,           -- 480 (value when goal was created)
  deadline TEXT NOT NULL,                  -- '2026-05-01' (required — premise #1)
  status TEXT NOT NULL DEFAULT 'active',  -- 'active' | 'completed' | 'expired' | 'paused'
  created_by TEXT NOT NULL DEFAULT 'user', -- 'user' | 'ai'
  oml_impact REAL,                       -- estimated OML points gained when complete
  completed_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE TABLE goal_progress_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  goal_id INTEGER NOT NULL REFERENCES goals(id),
  value REAL NOT NULL,
  logged_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### AI Coach Integration

**Provider note:** The app uses OpenRouter (OpenAI-compatible API) which routes to various models. The existing `src/services/ai.ts` already handles this — no new API integration needed.

**Failure modes for goal JSON parsing:**
- If the goals JSON block is **absent** from the AI response: no goal actions are taken, response displayed normally. This is expected behavior — not every response needs goal changes.
- If the goals JSON is **malformed**: error logged silently (`console.warn`), response displayed without crashing. No goal actions applied.
- If AI Coach is on but no goal action received for **5+ consecutive responses**: show a subtle banner "AI Coach is thinking about your goals..." to confirm it's still active.

**5-goal cap enforcement:** The `goalEngine.ts` (client-side) is the authoritative enforcer, NOT the LLM. If a `create` action would exceed 5 active goals, the action is silently discarded and a toast shows "Goal limit reached — complete or remove a goal first." The LLM's system prompt instruction is a nudge to reduce wasted API output, not a guarantee.

When AI Coach mode is enabled (stored in settings), the system prompt includes:

```
GOAL MANAGEMENT:
You are managing the cadet's goals. After your text response, output a JSON block
wrapped in ```goals tags with goal actions:

{
  "actions": [
    {"type": "create", "title": "...", "category": "acft", "metric": "acft_total",
     "target_value": 550, "deadline": "2026-05-01", "oml_impact": 12},
    {"type": "update", "goal_id": 3, "current_value": 530},
    {"type": "complete", "goal_id": 2, "message": "Great work!"},
    {"type": "retire", "goal_id": 4, "message": "Reprioritizing based on your progress."}
  ]
}

Action types:
- create: adds a new goal (status='active')
- update: updates current_value on an existing goal
- complete: marks goal as completed (status='completed')
- retire: marks goal as paused/deprioritized (status='paused')

Rules:
- Never exceed 5 active goals. If at 5, complete or retire one before adding.
- Prioritize goals by marginal OML gain (from the marginal gains data in context).
- Set deadlines based on the cadet's year group and branch selection timeline.
- When a goal is close to completion (>90%), encourage the final push.
- When a goal's deadline passes without completion, mark it expired with encouragement.
```

The AI service parses this JSON block from the response, strips it from the displayed text, and applies the goal actions to SQLite.

### Auto-Update Flow

```
Cadet logs new ACFT score (545)
  → SQLite: acft_assessments table updated
  → Goal engine: scan active goals where category='acft'
  → Found: "Improve ACFT to 550" (current: 530)
  → Update: current_value = 545
  → If current_value >= target_value → status = 'completed', celebrate!
  → Context Engine: include updated goals in next AI call
  → AI briefing: "Your ACFT goal is 91% complete — 5 more points!"
```

### Goal Card Design

Each goal card (VGoalCard component) shows:
- Category icon (💪/📚/🎖️/📊) + title
- Progress bar (gold gradient fill, same as pillar bars)
- Current/target values + delta
- Deadline + estimated OML impact
- Created-by badge: "AI" (olive chip) or "You" (no badge)
- Tap → goal detail: history of progress, AI insights about this goal

### Completion Celebration

When current_value >= target_value:
1. Goal card pulses gold (tertiary color glow animation)
2. Status changes to "completed"
3. Show OML delta: "🏆 Goal achieved! +12 OML points earned"
4. The AI's next briefing acknowledges: "You hit your ACFT goal. Here's your next challenge..."
5. Completed goals move to a "Completed" section (collapsed by default)

### Settings

Add to Settings screen:
- **AI Coach Mode:** toggle (default: off)
- When toggled on: "Vanguard AI will create and manage goals based on your profile. Max 5 active goals."
- When toggled off: existing goals remain but AI won't create new ones
- **Activation flow:** When the toggle is first enabled, the next AI briefing (whether automatic on app foreground or user-initiated chat) includes the goal management system prompt and will create initial goals based on the cadet's profile

## Open Questions

1. Should the AI be able to modify goals the user created manually? (Risk: feels like overstepping)
2. Should expired goals auto-delete or persist as "missed"?
3. Should there be a "suggested goals" queue where AI suggestions wait for acceptance even in Coach mode? (Belt-and-suspenders approach)

- Completing a goal shows the gold celebration animation
- The AI references active goals in its briefing and chat responses
